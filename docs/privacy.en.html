Privacy Policy

Image Prompt Reverser · Chrome Extension
Last updated: April 19, 2026 · Version 1.0
TL;DR: This extension does not collect, upload, or store any of your data on any developer server. All settings are stored only in your local browser, and images are sent only to the API endpoint you configure yourself.

1. No developer-operated servers

"Image Prompt Reverser" is an open-source Chrome extension that runs entirely in your local browser. The developer operates no backend service and therefore cannot receive, view, or store any of your data.

2. What the extension stores

The extension stores the following information only in your local browser, using Chrome's official chrome.storage.sync and chrome.storage.local APIs:

  • API Base URL — the model endpoint you configure
  • API Key — your credential
  • Model name — the vision model ID you configure
  • Default prompt-style preset (General / Nano Banana / Jimeng / Flux / SD)
  • A cache of models that don't support response_format (just model name strings, used to avoid redundant retries)

None of this data is transmitted to the developer or any third party. You can clear it at any time via the 🗑 Reset button on the extension's options page.

3. How images and prompts flow

When you trigger a reverse-prompt action (right-clicking an image or taking a screenshot), data flows as follows:

  • The image (as an URL or base64 screenshot) is sent directly to the API endpoint you configured (e.g. https://api.openai.com/v1 or any gateway of your choice)
  • The JSON result returned by the model is displayed in a floating panel, and is not saved by the extension
  • Once the panel is closed, the result is discarded
  • The developer has no access to any images or results you process

4. Permissions requested and why

  • contextMenus — to add "Reverse Prompt" entries to the right-click menu
  • storage — to locally save your configuration (see Section 2)
  • activeTab — to capture a screenshot of the current tab only when you click the extension icon
  • host_permissions: <all_urls> — to let you right-click images on any site and to render the result panel overlay. The extension does not read or upload page content.

5. What the extension does NOT do

This extension does not:

  • Collect or upload your browsing history
  • Read or transmit cookies
  • Integrate any third-party analytics or tracking (no Google Analytics, Sentry, etc.)
  • Access saved passwords, autofill data, or payment information
  • Make any API calls in the background — requests fire only when you actively trigger them

6. Third-party services

The extension only sends requests to the API endpoint you configure (OpenAI, Google Gemini, Anthropic Claude, any OpenAI-compatible gateway, etc.). Please review the privacy policy of the API provider you choose — how they handle image data is entirely up to them and unrelated to this extension.

7. Children's privacy

This extension is not directed to children under 13 and does not knowingly collect any data from them.

8. Data retention and deletion

Because all data is stored only in your local browser, you have full control. You can clear it by:

  • Clicking the 🗑 Reset button in the options page
  • Removing the extension from chrome://extensions/
  • Clearing your browser data

9. Changes to this policy

Any updates will be noted in the GitHub repository with a revised "Last updated" date. Material changes will also be mentioned in the extension's release notes.
